# my-claude-skills

Personal library of Claude agent skills. Each skill teaches Claude a specialized workflow.

## Structure

```
my-claude-skills/
├── src/                        # Source — edit these
│   └── skill-name/
│       └── SKILL.md
├── dist/                       # Packaged — generated by build.sh
│   └── skill-name.skill
├── build.sh                    # Package src/ → dist/
├── install.sh                  # Copy dist/ → Claude's skills directory
└── README.md
```

## Skills

| Skill | Description |
|---|---|
| [project-summarizer](src/project-summarizer/SKILL.md) | Generates and maintains a PROJECT.md for any JS/TS codebase |

## Workflow

### Add a new skill
```bash
mkdir src/my-new-skill
# Create src/my-new-skill/SKILL.md
./build.sh my-new-skill
./install.sh my-new-skill
```

### Update an existing skill
```bash
# Edit src/skill-name/SKILL.md
./build.sh skill-name
./install.sh skill-name
```

### Build and install everything
```bash
./install.sh --build
```

### On a new machine
```bash
git clone <this-repo>
cd my-claude-skills
chmod +x build.sh install.sh

# First: set your Claude skills path (see install.sh header if unsure)
export CLAUDE_SKILLS_DIR="$HOME/.claude/skills"

./install.sh --build
```

## Finding Your Claude Skills Directory

The install path isn't officially documented yet. Things to try:
- `~/Library/Application Support/Claude/skills` (macOS)
- `~/.config/claude/skills` (Linux)
- `~/.claude/skills`

Once you find it, set it permanently in your shell profile:
```bash
export CLAUDE_SKILLS_DIR="/path/to/claude/skills"
```

Or just hardcode it at the top of `install.sh`.

## Versioning Convention

Skills are versioned via git — no manual version numbers needed. To see the history of a skill:
```bash
git log --oneline -- src/project-summarizer/SKILL.md
```
